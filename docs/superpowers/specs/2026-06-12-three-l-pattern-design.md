# 三 L 拼铺装方案定义

日期：2026-06-12

## 目的

本文记录“三 L 拼”木地板铺装方案的精确定义，用于后续实现铺装生成算法。该定义来自坐标模型，而不是视觉近似。

## 基本参数

设木板为矩形：

- 板宽：`m`
- 板长：`n`
- 坐标系：二维直角坐标系，使用数学坐标，`x` 向右，`y` 向上
- 一块木板用左下角和右上角表示：`{(x1,y1),(x2,y2)}`

不考虑板缝，仅描述几何铺装关系。

## 基本单元

三 L 拼由两类铺装组构成：`A` 和 `B`。

### A 单元

`A` 单元由三块竖向木板组成，每块尺寸为 `m × n`。

若 `A1.a1` 的起点为 `(0,0)`，则：

```text
a1 = {
  {(0,0),(m,n)},
  {(m,0),(2m,n)},
  {(2m,0),(3m,n)}
}
```

也就是说，A 是三块并排的竖板，整体外接矩形为 `3m × n`。

### B 单元

`B` 单元由三块横向木板组成，每块尺寸为 `n × m`。

若 `B1.b1` 的起点为 `A1.a1 + (3m,0) = (3m,0)`，则：

```text
b1 = {
  {(3m,0),(3m+n,m)},
  {(3m,m),(3m+n,2m)},
  {(3m,2m),(3m+n,3m)}
}
```

也就是说，B 从 A 的右侧开始，三块横板上下堆叠，整体外接矩形为 `n × 3m`。

## 组内递推

同一组内部，后续单元沿向量 `d` 平移：

```text
d = (3m, 3m)
```

因此：

```text
a(k) = a1 + k·(3m,3m)
b(k) = b1 + k·(3m,3m)
```

其中 `k` 为整数。若只考虑单方向增长，可取 `k >= 0`；若要铺满无限平面，应允许 `k` 取所有整数。

## 组间递推

相邻组之间，起点沿向量 `u` 平移：

```text
u = (3m+n, 3m-n)
```

因此：

```text
A_r.a1 起点 = r·(3m+n, 3m-n)
B_r.b1 起点 = r·(3m+n, 3m-n) + (3m,0)
```

其中 `r` 为整数。若只向一个方向增长，可取 `r >= 0`；若要铺满无限平面，应允许 `r` 取所有整数。

## 完整平面生成公式

对任意整数 `r` 和 `k`，定义基点：

```text
P(r,k) = r·(3m+n, 3m-n) + k·(3m,3m)
```

则该基点上的 A 单元为：

```text
A(r,k) = {
  {P(r,k) + (0,0),  P(r,k) + (m,n)},
  {P(r,k) + (m,0),  P(r,k) + (2m,n)},
  {P(r,k) + (2m,0), P(r,k) + (3m,n)}
}
```

该基点上的 B 单元为：

```text
B(r,k) = {
  {P(r,k) + (3m,0),  P(r,k) + (3m+n,m)},
  {P(r,k) + (3m,m),  P(r,k) + (3m+n,2m)},
  {P(r,k) + (3m,2m), P(r,k) + (3m+n,3m)}
}
```

完整三 L 拼铺装为所有 `A(r,k)` 与 `B(r,k)` 的集合，其中 `r,k ∈ Z`。

## 面积与周期性

两个生成向量为：

```text
d = (3m, 3m)
u = (3m+n, 3m-n)
```

其平行四边形面积为：

```text
|det(u,d)| = |(3m+n)·3m - (3m-n)·3m| = 6mn
```

一个 `A+B` 基本单元的面积为：

```text
A 面积 = 3mn
B 面积 = 3mn
A+B = 6mn
```

因此，该定义在无限平面上是面积守恒的周期铺装模型。后续实现时，可以先生成足够大的整数索引范围，再用房间多边形裁剪。

## 示例：m=5, n=35

当板宽 `m=5`，板长 `n=35` 时：

```text
a1 = {
  {(0,0),(5,35)},
  {(5,0),(10,35)},
  {(10,0),(15,35)}
}
```

```text
a2 = a1 + (15,15) = {
  {(15,15),(20,50)},
  {(20,15),(25,50)},
  {(25,15),(30,50)}
}
```

```text
b1 = {
  {(15,0),(50,5)},
  {(15,5),(50,10)},
  {(15,10),(50,15)}
}
```

```text
b2 = b1 + (15,15) = {
  {(30,15),(65,20)},
  {(30,20),(65,25)},
  {(30,25),(65,30)}
}
```

组间平移向量为：

```text
u = (3m+n, 3m-n) = (50, -20)
```

## 现实板间缝扩展

上面的数学定义用于确认三 L 拼骨架，等价于板间缝 `g=0`。现实铺装需要考虑板间缝时，保留每块板的真实尺寸 `m × n` 或 `n × m`，但将坐标步距从 `m` 替换为：

```text
S = m + g
```

因此：

- A 三块竖板的 x 起点为 `0, S, 2S`
- B 三块横板的起点为 `(3S,0), (3S,S), (3S,2S)`
- 组内平移为 `(3S, 3S)`
- 组间平移为 `(3S+n, 3S-n)`

当 `g=0` 时，该扩展退化回原始数学模型。

## 实现注意事项

- 工程内部如果使用 SVG 屏幕坐标，需要注意 SVG 的 `y` 轴向下，而本文定义的数学坐标 `y` 轴向上。
- A 必须是竖向三板，尺寸为 `m × n`；B 必须是横向三板，尺寸为 `n × m`。
- 不应把该模式实现成棋盘式三横三竖交替。
- 不应把该模式实现成三三人字拼或斜向人字拼。
- 后续房间裁剪时，应从无限平铺中选取覆盖房间包围盒的有限索引范围，再对每块板与可铺区域求交。
- 现实铺装应使用 `board_gap` 生成 `S=m+g` 的坐标步距；不要用 `board_gap=0` 的数学骨架直接代表施工图。
