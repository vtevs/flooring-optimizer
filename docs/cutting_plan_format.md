# Cutting Plan 输出格式

`cutting_plan.txt` 是铺装优化后的切割清单。当前输出支持单房间和多房间；多房间时会按房间分组列出完整板、切割板、复用余料和损耗统计。

## 文件头

```text
=== 地板切割清单 ===
生成时间: {timestamp}
房间尺寸: {width} x {length} mm
地板规格: {board_length} x {board_width} x {board_thickness} mm
铺装方式: {pattern}  方向: {direction}°
```

多房间输出还会包含：

```text
=== 房间: {room_name} ===
房间尺寸: {width} x {length} mm
```

## 完整板

完整板表示一整片原板直接铺入，没有从其他板切割获得，也没有产生长度或宽度方向废料。

判断口径：

- `parent_board_id` 为空；
- `cut_pieces` 只有 1 片；
- 长度废料与宽度废料都为 0；
- 铺装宽度接近原板宽度。

输出示例：

```text
完整板 #12: 600.0 x 88.0 mm at (1200.0, 176.0)
```

## 切割板

切割板表示从一片原板切出的主要铺装片。切割板会显示尺寸、位置、切割方向和来源板号。

```text
切割板 #34: 512.5 x 88.0 mm at (2400.0, 352.0)
  来源: 原板 #34
  切割: length_cut
```

常见切割方向：

| 值 | 含义 |
| --- | --- |
| `none` | 无切割 |
| `length_cut` | 沿长度方向截短 |
| `width_cut` | 沿宽度方向裁窄 |
| `both_cut` | 长宽都切 |

## 余料复用

如果某块铺装片来自之前切割产生的余料，会输出父来源信息：

```text
复用余料 #81: 210.0 x 88.0 mm at (3000.0, 440.0)
  来源余料: #34:length_waste
```

复用余料仍按实际铺入面积计入覆盖面积；没有被复用的余料计入切割损耗。

## 损耗统计

```text
=== 损耗统计 ===
总采购面积: 31.0464 m²
可铺装面积: 30.1051 m²
有效铺装面积: 30.1051 m²
总损耗面积: 0.9413 m²
损耗率: 3.03%
```

损耗率口径：

```text
(总采购面积 - 有效铺装面积) / 总采购面积
```

其中有效铺装面积为房间 polygon 减去柜子等障碍物，再扣除四周伸缩缝后的可铺装区域面积。板缝用于相邻板间距和碰撞校验，不作为另一个独立铺装目标面积扣减。

## 注意事项

- 当前三L拼现实输出需要考虑 `board_gap`、`expansion_gap`、切割锯缝 `kerf` 和余料复用。
- 柜子、衣柜等固定家具以 `obstacles` 描述，不铺地板；伸缩缝预留到障碍物边缘。
- 榫槽约束以 `docs/flooring_constraints.md` 为准，内部邻边必须满足公榫/母榫对接；切割边不得作为内部对接边。
