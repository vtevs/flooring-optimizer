# 木地板铺装约束定义

日期：2026-06-16

本文记录当前项目采用的铺装、切割、榫槽、拼缝、伸缩缝和不可铺装区域约束。后续优化、校验、审计报告应以本文为准。

## 坐标与单位

- 坐标单位为 `mm`。
- 平面坐标以房间左下角为 `(0,0)`。
- `x` 轴向右，`y` 轴向上。
- 用户描述中，“长”为 `X` 方向，“宽”为 `Y` 方向。
- SVG 渲染可以使用屏幕坐标翻转，但几何计算必须保持上述数学坐标。

## 单片地板规格

当前项目地板规格为：

```text
长度: 600mm
宽度: 88mm
厚度: 18mm
```

铺装几何中，地板被视为平面矩形；厚度只用于约束是否允许翻面。

## 榫槽基准定义

以一片地板“板面朝上、长边水平放置”为基准：

```text
左长边 = 母榫 GROOVE (G)
右长边 = 公榫 TONGUE (T)
下短边 = 母榫 GROOVE (G)
上短边 = 公榫 TONGUE (T)
```

即基准四边为：

```text
top    = TONGUE
bottom = GROOVE
left   = GROOVE
right  = TONGUE
```

## 安装方向约束

- 不允许沿厚度方向翻面使用，即 18mm 的顶部和底部不能互换。
- 允许在俯视平面内旋转使用。
- 每块板在校验中允许选择以下四种平面旋转状态：
  - `0°`
  - `90°`
  - `180°`
  - `270°`

旋转只改变四边在平面坐标中的对应方向，不改变板面朝上这一事实。

## 邻边拼接定义

内部邻边是指两块地板在平面中：

- 边互相平行；
- 边之间的距离约等于 `board_gap`；
- 两条边在投影方向上有有效重叠；
- 该边不是房间边界、柜子边界或其它不可铺装区域边界。

内部邻边必须满足：

```text
TONGUE ↔ GROOVE
GROOVE ↔ TONGUE
```

不允许：

```text
TONGUE ↔ TONGUE
GROOVE ↔ GROOVE
CUT    ↔ 任意内部邻边
```

其中 `CUT` 表示切割面。切割面只能出现在房间边界、柜子边界、L 形缺口边界等非内部拼接边。

## 公母榫严格证明

校验器对每个房间执行以下证明：

1. 从铺装结果中提取每块板的实际使用多边形。
2. 根据 `board_gap` 识别所有内部邻边。
3. 若内部邻边包含切割边，则立即判定失败。
4. 对每块板建立四种可选平面旋转状态：`0/90/180/270`。
5. 用约束求解证明所有内部邻边可以同时满足 `T/G` 对接。

若求解成功，则认为该房间的公母榫对接被严格证明。

## 板间拼缝

当前三 L 拼现实铺装使用：

```text
board_gap = 0.5mm
```

`board_gap` 进入三 L 拼坐标步距：

```text
S = board_width + board_gap
```

板间拼缝不消耗木材，但会占据可铺区域面积。因此审计报告同时给出两种损耗口径：

- 显示口径：`可铺区域面积 / 采购整板面积`
- 木材口径：`实际木材覆盖面积 / 采购整板面积`

## 伸缩缝

当前伸缩缝为：

```text
expansion_gap = 10mm
```

伸缩缝作用于所有需要预留的边界：

- 房间墙边；
- L 形房间内角和外边；
- polygon 房间轮廓边；
- 柜子、衣柜、固定家具等不可铺装区域边缘。

几何计算顺序为：

```text
房间原始轮廓 - 不可铺装区域 = 净房间区域
净房间区域 buffer(-expansion_gap) = 最终可排样区域
```

踢脚线宽度当前不扣减可铺区域；代码注释假定踢脚线压在地板上方。

## 不可铺装区域

不可铺装区域包括柜子、衣柜、固定家具等。它们在配置中通过 `obstacles` 描述。

矩形障碍物示例：

```yaml
obstacles:
  - name: wardrobe
    type: rectangle
    x: 0
    y: 2862
    width: 1830
    length: 600
```

多边形障碍物示例：

```yaml
obstacles:
  - name: custom-cabinet
    type: polygon
    points:
      - [1000, 500]
      - [1600, 500]
      - [1700, 900]
      - [1200, 1100]
```

渲染规则：

- 障碍物区域以矩形或多边形显示；
- 内部画两条对角线形成 `X`；
- 地板不得铺入障碍物区域，也不得铺入障碍物边缘预留的伸缩缝区域。

## 房间轮廓

当前支持以下房间类型：

### rectangle

```yaml
type: rectangle
width: 2856
length: 3462
```

### l-shaped

表示外接矩形减去一个角部缺口：

```yaml
type: l-shaped
width: 3600
length: 4200
notch:
  position: top-left
  width: 1200
  depth: 900
```

### polygon

用于描述凸出、凹入或更一般的房间轮廓：

```yaml
type: polygon
points:
  - [0, 0]
  - [3254, 0]
  - [3254, 3460]
  - [1063, 3460]
  - [1063, 5447]
  - [0, 5447]
```

点按顺时针或逆时针给出均可，但不能自交。

## 切割与锯缝

当前锯缝为：

```text
kerf = 1.0mm
```

切割约束：

- 长度切割产生长度尾料，尾料可复用；
- 宽度切割产生宽度尾料，满足可用条件时可复用；
- 组合切割同时记录长度废料和宽度废料；
- 每次切割都应满足：使用片面积 + 显式废料面积 + 锯缝面积 ≈ 原料面积；
- 锯缝不单独计入采购面积，而是通过可复用尺寸和废料面积影响最终用板数。
