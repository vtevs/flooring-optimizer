## 2. 核心算法方向

### 2.1 对拼（直铺）

最简单模式：所有地板同向平行排列，行与行之间无偏移。

算法：
- 遍历起始偏移（0 ~ 板宽），选择覆盖面积最大的
- 每行 = 完整板 + 尾端切割板
- 优化变量：铺装方向（平行长边 vs 平行短边）、起始偏移

**榫槽约束**：由于地板有榫槽方向，切割后的剩余板只能用在同一方向，不能翻转 180° 后接在另一端。这意味着每行的切割尾料只能在当前方向的行内复用。

### 2.2 错缝（Staggered）

与对拼类似，但行间有水平偏移。偏移量 = 板长 × 错缝比例（可配置，如 1/3、1/2）。

算法：
- 固定错缝比例后，枚举起始偏移
- 每行起始端可能需要切割以满足错缝对齐
- 优化变量：铺装方向、错缝比例、初始行偏移

**行业常见错缝比例**：1/3（≈33%）和 1/2（50%），但支持完全自定义。

### 2.3 人字拼（Herringbone 45°）

经典 45° 人字拼：
- 每块板与墙面成 45° 角
- 相邻板垂直交错，形成 V 形图案
- 本质是 45° 旋转后的平行四边形网格密铺

算法：
1. 建立人字拼密铺网格（两块相邻板 = 一个密铺单元，呈平行四边形）
2. 将房间多边形旋转 45° 后与网格求交
3. 完全在房间内的板保留原样；跨边界的板按房间边界裁剪
4. 边界裁剪产生三角形/梯形废料

**关键约束**：榫槽方向在人字拼中更复杂，因为相邻板方向垂直，需要确认拼接是否可行。

### 2.4 5拼方砖

5 块相同规格的板并排组成一个正方形单元。例如 440×88mm 的板，5 块并排 = 440×440mm 正方形。

排列规则：
- 单元 A：板横向排列（板的长边水平）
- 单元 B：板纵向排列（板的长边垂直）
- 单元 A 和 B 交替排列（棋盘式）

```
单元 A (横排):        单元 B (竖排):
┌─────────────────┐   ┌──┬──┬──┬──┬──┐
│      ←440→     │   │↑│  │  │  │  │
├─────────────────┤   │4│  │  │  │  │
│      ←440→     │   │4│  │  │  │  │
├─────────────────┤   │0│  │  │  │  │
│      ←440→     │   │↓│  │  │  │  │
├─────────────────┤   ├──┼──┼──┼──┼──┤
│      ←440→     │   │  │  │  │  │  │
├─────────────────┤   │  │  │  │  │  │
│      ←440→     │   │  │  │  │  │  │
└─────────────────┘   └──┴──┴──┴──┴──┘
 (5块 440×88 横排)     (5块 440×88 竖排)
```

算法：
1. 将正方形单元视为不可分割的虚拟大板
2. 按棋盘网格密铺房间
3. 边界处的单元整体裁切（单元内每块板独立裁剪）
4. 横竖交替的棋盘由单元坐标 (row+col) % 2 决定方向

### 2.5 统一的损耗优化框架

由于地板有**榫槽方向约束**，切割剩余板不能随意翻转使用：

```
可排样区域 = 房间多边形 - 踢脚线内缩 - 四周伸缩缝
```

优化流程：
1. 定义可排样区域
2. 对每种铺装方式，枚举起始偏移（网格滑动窗口）
3. 模拟排样过程，记录每块板的状态（完整/切割/废料）
4. 计算损耗率 = 废料面积 / 总用料面积
5. 选择损耗率最低的方案

## 3. 参数规格确定

### 3.1 伸缩缝（Expansion Gap）

**地暖房推荐值：10-12mm 四周伸缩缝**

| 场景 | 推荐值 | 说明 |
|------|--------|------|
| 普通房间 | 8-10mm | 国标 GB/T 20238-2018 |
| 地暖房间 | 10-12mm | 地暖温差大，需更大伸缩余量 |
| 大面积 (>100㎡) | 12-15mm | 还需考虑中部伸缩缝 |

**默认值取 10mm**（地暖通用），可在配置文件中覆盖。

板间留缝（板与板之间的微缝）：通常 0.3-1mm，可配置，默认 0。

### 3.2 榫槽方向约束

地板有公榫和母槽，决定了拼接方向：
- 每块板的四个边中，两边为公榫，两边为母槽
- 板只能沿榫槽方向插入邻板
- 这意味着**切割后的板不能翻转 180° 直接使用**，因为翻转后榫槽位置不对

实际影响：
- 某行切下的尾料（有榫的一侧）只能用于下一行同侧的起始
- 不能把尾料翻转后用于另一侧

## 7. 多房间优化与共享尾料池

### 7.1 核心思路

多个房间共用一批地板时，房间 A 的切割尾料可用于房间 B 的行首，减少总用板量。

### 7.2 实现架构

```
multi_optimize.py → optimize_multi()
  Phase 1: 每房间独立找最优配置
  Phase 2: 独立方案作为基线
  Phase 3: 枚举房间顺序 + 共享 BoardPool → 比较取最优
```

### 7.3 实测结果

**3 房间案例** (对拼 90° + 错缝):

| 方案 | 用板 | 利用率 | 说明 |
|------|------|--------|------|
| 独立优化 | 218 | ~99% | 每房间单独优化 |
| 共享池(初版) | 280 | 74% | 池优化目标错误（最少新板而非利用率） |
| 共享池(修正后) | — | — | 修正目标后共享池未改善此案例 |
| 最终方案 | 216 | 100.1% | 独立优化 + 微调 = 接近理论下限 (216.3) |

**关键发现**：共享池不总是有益的。当各房间错缝产生的尾料尺寸（如 ~455mm, ~227mm）不匹配其他房间间隙需求时，共享池无改善。**同户型多个相同尺寸房间**是最佳共享池场景。

### 7.4 设计决策

- **比较策略**：独立方案作为基线，共享池方案仅当更优时采用
- **全局 ID**：多房间切割方案需要跨房间唯一源板编号，便于追溯
- **引擎透明**：引擎通过可选 `pool=` 参数接收共享池，不影响单房间模式

### 7.5 共享池理论优势场景

- 同一户型多个房间尺寸完全相同 → 尾料尺寸系统性地匹配
- 对拼模式（非错缝）→ 端头切割产生统一尺寸尾料
- 大房间 + 小房间组合 → 大房间余料填小房间
